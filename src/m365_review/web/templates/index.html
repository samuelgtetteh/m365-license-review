Output formats

You'll be sent to Microsoft, sign in as a client tenant admin, and approve read-only consent. The tool never writes to the tenant.

What the audit checks

R1

Disabled users with licenses

Licenses still assigned to disabled accounts — pure waste.

R2

Inactive users (90 days)

Licensed, enabled users with no sign-in in 90 days. Degrades gracefully without Entra P1.

R3

Unassigned licenses

Purchased seats nobody is using — reclaim at renewal.

R4

Duplicate / overlapping

Users holding a SKU a superset license already covers (e.g. E5 + E3).

R8

Expiring subscriptions

Subscriptions expired or expiring within 30 days — renew, resize, or cancel.

R7

Licensed guests

Guest accounts holding licenses they usually don't need (they use their home tenant's).

R9

Users without MFA

Member accounts with no registered MFA method — a top account-takeover risk.

R10

Admin audit

Admins without MFA, and too many Global Administrators (least-privilege).

✓

Paid vs. free split

Free/self-service SKUs and inactive subscriptions are separated, not counted as spend.

R11

CA requires MFA (all users)

An enabled Conditional Access policy requires MFA for all users and apps.

R12

CA blocks legacy auth

An enabled CA policy blocks legacy authentication clients (which bypass MFA).

R15

Global-admin MFA coverage

Admins are covered by a CA MFA policy.

R13

Auth-methods policy

Strong methods enabled, weak/phishable methods (SMS, Voice) off.

R14

Trusted locations

Review named locations that can bypass controls.

R16

Per-user MFA state

Legacy per-user MFA state (fine if you enforce via Conditional Access).

R17

Verified domains

Review tenant domains; flag unverified or federated ones.

Pick exactly what to run on the Run audit tab. The Conditional Access, auth-policy and per-user-MFA checks need the read-only Policy.Read.All scope — the tool only requests it when you select one of them.

  • Optional “Prepared by …” company branding.

Prefer the command line?

The web app and CLI share one engine. This is the equivalent command for the options on the Run audit tab — copy it to script or automate.

m365-review run --format xlsx --format docx --format json

In the container (Rancher or Docker Desktop):

docker compose run --rm m365-review m365-review run --profile "Contoso" --tenant contoso.com
docker compose run --rm m365-review m365-review profiles list

See the Help page for the full CLI guide.

Read-only guarantee

  • Only read scopes are requested and only GET calls are made — enforced by a runtime guard.
  • Your access token stays in memory for the run only — never logged, saved, or put in a report.
  • Reports contain client PII — store them somewhere access-controlled.
